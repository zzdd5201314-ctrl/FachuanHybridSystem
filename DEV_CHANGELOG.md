# Dev 更新记录

用于记录本地 `dev` 分支准备推送的阶段性变更。

从 `main` 同步来的正式版本更新只记录索引，详细内容仍以 `CHANGELOG.md` 为准；本地 `dev` 自己新增或修复的功能在这里展开记录。

## 2026-05-13

### 后台案件

- 新增案件详情“重要时间”功能，支持手动添加、日志提醒同步、状态展示、删除/移除。
- 重要时间复用 `Reminder` 数据，不新增独立案件时间表。
- 增加重要时间相关测试覆盖，包含手动重要时间、日志同步、未勾选不展示、逾期状态等场景。

### 合同文件夹

- 修复合同绑定文件夹时设备号整数溢出问题。
- 修复合同文件夹绑定子目录越界提示乱码。

## 2026-05-12

### 从 main 同步

- 同步 `main` 上 `CHANGELOG.md` 已记录的 `26.48.6`、`26.48.7` 版本更新。
- 同步范围包含一张网立案/担保耗时统计、诉讼保全担保流程优化、SystemConfig 管理页优化、安全依赖和 GitHub Actions 依赖更新。

### 本地 dev 更新

- 日志功能：优化案件日志 inline 模板为紧凑展示，后台编辑页展示更集中。
- 日志功能：恢复案件日志和合同材料绑定相关后台能力，避免日志/材料入口缺失。
- Admin 后台：修复空客户身份文档 inline 导致的异常。
- 案件后台：合并案件相关并行迁移，避免迁移冲突。
- 合同后台：修复合同导出时的预取冲突问题。

## 2026-05-11

### 从 main 同步

- 同步 `main` 上 `CHANGELOG.md` 已记录的 `26.48.3`、`26.48.4`、`26.48.5` 版本更新。
- 同步范围包含消息来源和收件箱附件优化、模板编辑页重构、OpenAI 兼容 LLM 后端、批量分析下载和取消状态修复等正式版本内容。

### 本地 dev 更新

- 日志功能：案件日志列表增加编辑操作入口，方便从日志列表直接维护记录。
- 日志功能：案件详情页增加案件日志快捷入口，减少进入日志维护的跳转成本。
- 日志功能：修复案件日志保存子目录正则异常，避免空值或特殊路径导致崩溃。
- 附件功能：新增附件按业务文件夹保存能力，后续日志附件、材料附件可以进入更明确的业务目录。
- Admin 后台：收紧 inline 默认行为，减少空 inline 或异常 inline 影响后台页面。
